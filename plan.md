# 일정 공유 캘린더 (shared-calendar) — 앱 기획서

> session_id: `20260602-shared-calendar`
> 아이디어: 아이디/비밀번호 없이 세션을 공유하는 형태의 일정 공유 캘린더. 위젯 형태로 달력을 볼 수 있어야 함.

---

## 전제 가정

intake.json의 `ambiguities` 항목을 다음과 같이 합리적 best-guess로 확정한다. (MVP 기준)

| # | 모호점 | 확정 방침 |
|---|--------|-----------|
| A-1 | 세션 공유 방식 | **URL 토큰 기반.** 세션 생성 시 추측 불가능한 랜덤 토큰(예: 22자 nanoid)을 발급하고 `/{session-token}` 형태의 공유 URL을 제공한다. 별도 초대 코드 입력 단계는 두지 않는다. |
| A-2 | 세션 권한 모델 | **MVP는 단일 권한.** 링크를 가진 모든 사용자가 읽기·쓰기 모두 가능하다. 읽기 전용/편집 권한 구분은 Out of Scope (향후 확장). |
| A-3 | 세션 만료 정책 | **마지막 활동일 기준 90일 자동 만료.** 만료 시 세션과 종속 데이터(이벤트)는 소프트 삭제 후 30일 경과 시 하드 삭제한다. 영구 보존 옵션은 두지 않는다. |
| A-4 | 위젯 임베드 대상 | **외부 웹사이트 iframe 임베드.** `<iframe src="/embed/{session-token}">` 형태로 어디든 삽입 가능한 읽기 중심 임베드 위젯을 제공한다. OS 네이티브 위젯은 Out of Scope. |
| A-5 | 위젯 캘린더 뷰 단위 | **월간(기본) + 주간 지원.** 일간 뷰는 Out of Scope. 임베드 위젯은 월간 뷰 고정. |
| A-6 | 익명 사용자 식별 | **세션 입장 시 표시 이름 입력 + 자동 색상 배정.** 표시 이름은 브라우저 로컬에 저장하여 재방문 시 재사용한다. 색상은 이름 해시 기반 팔레트에서 배정한다. |
| A-7 | 실시간 동기화 | **MVP는 폴링 기반(30초 간격) 갱신.** WebSocket 실시간 동기화는 Out of Scope (향후 확장). |
| A-8 | 이벤트 속성 범위 | **제목·시작/종료 시각·종일 여부·설명·위치·색상.** 반복 일정 및 알림은 Out of Scope. |
| A-9 | 세션 간 격리 | **1 세션 = 1 캘린더.** 한 세션은 정확히 하나의 캘린더를 가진다. 멀티 캘린더는 Out of Scope. |

## 기능 요구사항 (FR)

- **FR-001** 사용자는 아이디/비밀번호 입력 없이 "새 캘린더 만들기"로 세션을 즉시 생성할 수 있다.
- **FR-002** 세션 생성 시 시스템은 추측 불가능한 랜덤 토큰과 공유 URL(`/{session-token}`)을 발급한다.
- **FR-003** 공유 URL을 가진 사용자는 추가 인증 없이 해당 세션 캘린더에 접근할 수 있다.
- **FR-004** 세션 입장 시 사용자는 표시 이름을 입력하며, 시스템은 이름 기반 색상을 자동 배정한다. 표시 이름은 브라우저에 저장되어 재방문 시 재사용된다.
- **FR-005** 사용자는 캘린더에 이벤트를 생성할 수 있다(제목, 시작/종료 시각, 종일 여부, 설명, 위치, 색상).
- **FR-006** 사용자는 기존 이벤트를 수정·삭제할 수 있다.
- **FR-007** 캘린더는 월간 뷰(기본)와 주간 뷰를 제공하며 두 뷰를 전환할 수 있다.
- **FR-008** 각 이벤트에는 작성자의 표시 이름과 색상이 함께 표시된다.
- **FR-009** 캘린더는 30초 간격 폴링으로 다른 사용자의 변경 사항을 자동 반영한다.
- **FR-010** 시스템은 임베드용 읽기 전용 위젯(`/embed/{session-token}`)을 제공하며 iframe으로 외부 사이트에 삽입 가능하다.
- **FR-011** 사용자는 공유 URL을 클립보드로 복사할 수 있는 "공유" 버튼을 사용할 수 있다.
- **FR-012** 사용자는 월/주 단위로 이동(이전/다음/오늘)할 수 있다.
- **FR-013** 시스템은 마지막 활동일로부터 90일이 경과한 세션을 자동 만료 처리한다.
- **FR-014** 존재하지 않거나 만료된 세션 토큰 접근 시 명확한 안내 화면을 표시한다.

## 비기능 요구사항 (NFR)

- **NFR-001 (보안)** 세션 토큰은 추측 불가능한 고엔트로피 랜덤 값(≥128bit)이어야 하며, 검색엔진 인덱싱을 차단한다(noindex). 인증이 없는 만큼 토큰 노출 = 접근 권한임을 전제로 설계한다.
- **NFR-002 (성능)** 월간 뷰 캘린더 초기 렌더 및 폴링 응답은 일반 회선에서 1초 이내 완료되어야 한다.
- **NFR-003 (가용성/내구성)** 이벤트 생성·수정 데이터는 손실 없이 영속 저장되며, 만료 데이터는 30일 유예(소프트 삭제) 후 하드 삭제한다.
- **NFR-004 (임베드 호환성)** 임베드 위젯은 주요 브라우저 iframe 환경에서 동작하며 CSP/X-Frame-Options를 임베드 허용에 맞게 구성한다.
- **NFR-005 (사용성)** 첫 사용자가 캘린더 생성 → 공유 → 이벤트 등록까지 별도 설명 없이 3분 이내 수행 가능해야 한다.

## 사용자 스토리

- **US-001** 사용자로서, 회원가입 없이 캘린더를 즉시 만들어 바로 사용을 시작하고 싶다.
- **US-002** 사용자로서, 생성된 공유 링크를 팀원에게 전달해 같은 캘린더를 보게 하고 싶다.
- **US-003** 팀원으로서, 링크만 클릭하면 로그인 없이 캘린더에 들어가고 싶다.
- **US-004** 사용자로서, 입장 시 내 이름을 정해 내가 만든 일정이 누구 것인지 구분되게 하고 싶다.
- **US-005** 사용자로서, 날짜를 골라 제목과 시간을 적어 일정을 등록하고 싶다.
- **US-006** 사용자로서, 잘못 등록한 일정을 수정하거나 삭제하고 싶다.
- **US-007** 사용자로서, 월간 뷰로 전체 흐름을 보고 주간 뷰로 상세를 보고 싶다.
- **US-008** 팀원으로서, 다른 사람이 방금 추가한 일정이 내 화면에도 곧 반영되길 원한다.
- **US-009** 사용자로서, 캘린더를 우리 팀 노션/사이트에 위젯으로 끼워 넣고 싶다.
- **US-010** 사용자로서, 공유 버튼 한 번으로 링크를 복사하고 싶다.
- **US-011** 사용자로서, 만료되었거나 잘못된 링크일 때 무엇이 문제인지 안내받고 싶다.
- **US-012** 사용자로서, 오늘/이전/다음 버튼으로 원하는 기간으로 빠르게 이동하고 싶다.

## MVP 범위

### In Scope
- 인증 없는 세션(캘린더) 생성 및 토큰 기반 공유 URL
- 표시 이름 입력 + 자동 색상 배정 (익명 사용자 식별)
- 이벤트 CRUD (제목/시작·종료 시각/종일/설명/위치/색상)
- 월간 뷰 + 주간 뷰, 기간 이동(이전/다음/오늘)
- 작성자 표시(이름·색상)
- 30초 폴링 기반 변경 반영
- iframe 임베드용 읽기 전용 위젯 (월간 고정)
- 공유 링크 복사
- 세션 90일 자동 만료 + 만료/무효 링크 안내

### Out of Scope (향후 확장)
- 아이디/비밀번호·소셜 로그인 등 정식 인증
- 읽기 전용/편집자 권한 분리 (A-2)
- WebSocket 실시간 동기화 (A-7)
- 반복 일정, 알림/리마인더 (A-8)
- 일간 뷰 (A-5)
- OS 데스크톱/모바일 네이티브 위젯 (A-4)
- 멀티 캘린더(한 세션 내 여러 캘린더) (A-9)
- 외부 캘린더 연동(Google/iCal), 파일 첨부, 댓글

## 핵심 엔티티

### Session
세션 = 캘린더 1개. 인증 없이 토큰으로 접근한다.

| 속성 | 타입 | 설명 |
|------|------|------|
| id | uuid (PK) | 내부 식별자 |
| token | string (unique) | 공유 URL용 고엔트로피 랜덤 토큰 |
| title | string | 캘린더 표시 제목 (기본값 자동 부여 가능) |
| createdAt | datetime | 생성 시각 |
| lastActiveAt | datetime | 마지막 활동 시각 (만료 계산 기준) |
| expiresAt | datetime | 만료 예정 시각 (lastActiveAt + 90일) |
| deletedAt | datetime (nullable) | 소프트 삭제 시각 |

### Event
세션(캘린더)에 속한 일정 항목.

| 속성 | 타입 | 설명 |
|------|------|------|
| id | uuid (PK) | 식별자 |
| sessionId | uuid (FK → Session) | 소속 세션 |
| title | string (required) | 일정 제목 |
| startAt | datetime (required) | 시작 시각 |
| endAt | datetime (required) | 종료 시각 |
| allDay | boolean (default false) | 종일 일정 여부 |
| description | string (nullable) | 설명 |
| location | string (nullable) | 위치 |
| color | string | 표시 색상 (작성자 색상 또는 지정 색상) |
| authorName | string | 작성자 표시 이름 |
| createdAt | datetime | 생성 시각 |
| updatedAt | datetime | 수정 시각 |

### Participant (경량 / 클라이언트 중심)
익명 사용자 식별 정보. MVP에서는 서버 영속화 없이 브라우저 로컬 저장 + 이벤트의 `authorName`/`color`로 표현한다. (서버 영속이 필요해지면 별도 테이블로 승격)

| 속성 | 타입 | 설명 |
|------|------|------|
| displayName | string | 표시 이름 (브라우저 로컬 저장) |
| color | string | 이름 해시 기반 자동 배정 색상 |

### 관계
- `Session 1 — N Event` (한 세션이 여러 이벤트를 가짐; 세션 = 캘린더 1개)
- `Participant`는 정규 엔티티가 아닌 클라이언트 식별 정보이며, 작성자 정보로서 `Event.authorName` / `Event.color`에 비정규화되어 저장된다.
